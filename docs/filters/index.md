# Filters

Filters are used to determine which users are selected for entering [workflows](../workflow/index.md) and transitioning
between [workflow steps](../workflow/steps.md).

Like [actions](../actions/index.md), filters are implemented as subplugins and can therefore be easily extended by
installing further filter subplugins or creating your own institution-specific filters. The following filter types are
part of the core plugin and therefore available within every installation:
