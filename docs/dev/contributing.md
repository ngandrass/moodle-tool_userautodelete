# Contributing

Thank you very much for your interest in improving this plugin :heart:

For bugs and feature requests, please use the [GitHub issue tracker](https://github.com/ngandrass/moodle-tool_userautodelete/issues).
For code contributions, open a pull request on [GitHub](https://github.com/ngandrass/moodle-tool_userautodelete/).

[:material-bug: Issue Tracker](https://github.com/ngandrass/moodle-tool_userautodelete/issues){.md-button}
&nbsp;&nbsp;&nbsp;
[:material-source-pull: Pull Requests](https://github.com/ngandrass/moodle-tool_userautodelete/pulls){.md-button}


## Pull request requirements

- Follow the Moodle coding style.
- Keep changes small, readable, and well documented.
- Add or update tests for behavior changes.
- Include complete PHPDoc for new classes, methods, and properties.


## Before you submit

- Run the relevant PHPUnit tests. See [Testing Instructions](testing.md).
- If you changed documentation, verify the docs build. See [Building the Documentation](builddocs.md).
- Verify language strings, capabilities, and upgrade steps when applicable.

Thank you for helping make this plugin better!


## Questions are always welcome

Please do not hesitate to reach out to the maintainers if you have any questions or want to contribute to the project.
We are happy to help and provide guidance for your contributions :)

Just open an issue over on the [GitHub issue tracker](https://github.com/ngandrass/moodle-tool_userautodelete/issues).
